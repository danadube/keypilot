# ShowingHQ Beta Plan

Internal document defining the external beta program for ShowingHQ.

---

## Who the beta is for

- **Real estate agents** who run open houses and want to capture visitor leads
- **Early adopters** willing to use a new product and share feedback
- **Target:** 10–25 agents for initial beta

---

## What ShowingHQ includes right now

- **Open house management** — Create and manage open houses with property, date, time
- **Visitor sign-in** — QR code and link for guests to check in on their phone
- **Contact capture** — Name, email, phone, lead status
- **Visitors dashboard** — View and search all sign-ins
- **Follow-up drafts** — AI-generated follow-up emails per visitor
- **Email sending** — Send follow-ups via Gmail integration (Resend)
- **Google Calendar** — Connect for scheduling context
- **Gmail** — Connect to send follow-up emails
- **Seller reports** — Basic report export for open house hosts

---

## What is intentionally not included yet

- **Billing or subscriptions** — Beta is free
- **SMS/call features** — Email focus only for now
- **Advanced CRM** — Full contact management lives in ClientKeep (future)
- **Mobile app** — Web-first; mobile-responsive sign-in
- **Bulk import** — Manual entry only
- **Custom branding** — KeyPilot/ShowingHQ branding only

---

## First actions for beta users

1. **Create your first open house** — Property, date, title
2. **Test the sign-in page** — Preview QR and link; submit a test sign-in
3. **Connect Gmail** — Required for sending follow-up emails
4. **Connect Google Calendar** — Optional but recommended
5. **Capture your first visitor** — Run a real open house or use a test sign-in
6. **Review and send a follow-up** — Generate drafts, edit, send

---

## What success looks like in beta

- Agent completes all six first actions above
- Agent captures at least 1 real visitor (beyond test)
- Agent sends at least 1 follow-up email
- Agent provides feedback via in-product mechanism
- No critical blockers to core flow (sign-in → visitor → follow-up)

---

## Feedback collection

Beta users can submit feedback via:

- **In-product** — "Send feedback" in the sidebar (when in ShowingHQ)
- **Types:** Bug report, Feature request, General feedback

---

## Beta duration

- **Initial phase:** 4–6 weeks
- **Review:** Gather feedback, fix critical issues, refine onboarding
- **Next:** Transition to paid beta or general availability
