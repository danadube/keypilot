# KeyPilot — AI Workflow Rules

When completing development tasks for KeyPilot:

1. **Always build before finishing** — Run `npm run build` (or `npm run validate`) to confirm the app compiles.
2. **Always commit and push** — Do not leave work in local uncommitted state.
3. **Use clear commit messages** — Prefer prefixes: `feature:`, `fix:`, `ui:`, `refactor:`, `docs:`, `chore:`.
4. **Summarize the final result** — List files changed, build result, commit message, and push result.

Every completed task must end with:
1. Build check
2. Commit
3. Push

See `docs/dev-workflow.md` for full workflow and GitHub/Vercel setup.
